# 更新库存接口优化说明

**优化日期：** 2025-11-04  
**修改内容：** 将更新库存接口从 Query Parameter 改为 Request Body

---

## 🔧 修改内容

### 修改前

**接口定义：**
```java
@PutMapping("/{productId}/stock")
public Result<Void> updateStock(
        @PathVariable Long productId,
        @RequestParam Integer stock) {
    // ...
}
```

**请求方式：**
```
PUT /api/product/1/stock?stock=100
```

**缺点：**
- ❌ 参数在 URL 中，不安全
- ❌ 无法添加参数验证
- ❌ 难以扩展（添加其他参数）
- ❌ 不符合 RESTful 规范（修改操作应使用 Request Body）

---

### 修改后

**新建 UpdateStockDTO：**
```java
@Data
@Schema(description = "更新库存请求")
public class UpdateStockDTO {

    @Schema(description = "库存数量", example = "100")
    @NotNull(message = "库存数量不能为空")
    @Min(value = 0, message = "库存数量不能为负数")
    private Integer stock;
}
```

**接口定义：**
```java
@PutMapping("/{productId}/stock")
public Result<Void> updateStock(
        @PathVariable Long productId,
        @Valid @RequestBody UpdateStockDTO updateStockDTO) {
    log.info("更新库存：productId={}, stock={}", productId, updateStockDTO.getStock());
    productService.updateProductStock(productId, updateStockDTO.getStock());
    return Result.success("库存更新成功");
}
```

**请求方式：**
```
PUT /api/product/1/stock
Content-Type: application/json

{
  "stock": 100
}
```

**优点：**
- ✅ 参数在 Request Body 中，更安全
- ✅ 支持参数验证（`@NotNull`, `@Min`）
- ✅ 易于扩展（可添加 `reason`, `operator` 等字段）
- ✅ 符合 RESTful 规范
- ✅ Swagger 文档更清晰

## 🎯 参数验证说明

### @NotNull

- **作用：** 确保参数不为 null
- **错误提示：** "库存数量不能为空"

### @Min(value = 0)

- **作用：** 确保库存数量 ≥ 0
- **错误提示：** "库存数量不能为负数"

### @Valid

- **作用：** 启用 DTO 中的验证注解
- **位置：** Controller 方法参数前

## 🎓 RESTful API 最佳实践

### PUT 请求规范

**PUT 请求应该：**
- ✅ 使用 Request Body 传递数据
- ✅ 用于更新资源
- ✅ 幂等性（多次请求结果相同）

**不推荐：**
- ❌ 在 URL 中传递敏感数据
- ❌ 使用 Query Parameter 传递修改数据

### URL 设计规范

```
✅ 好的设计：
PUT /api/product/1/stock
Body: {"stock": 100}

❌ 不好的设计：
PUT /api/product/1/stock?stock=100
```
