# WebShop 后端快速启动指南

## 环境要求

- **JDK**: 21
- **Maven**: 3.6+
- **MySQL**: 8.0+
- **IDE**: IntelliJ IDEA / Eclipse（推荐 IDEA）

## 启动步骤

### 第一步：配置数据库

1. 启动 MySQL 数据库服务

2. 创建数据库（可选，如果尚未创建）：
```sql
CREATE DATABASE IF NOT EXISTS web_shop 
CHARACTER SET utf8mb4 
COLLATE utf8mb4_unicode_ci;
```

3. 修改 `src/main/resources/application.properties` 中的数据库密码：
```properties
spring.datasource.password=你的MySQL密码
```

### 第二步：安装依赖

在项目根目录执行：
```bash
mvn clean install
```

或者在 IDEA 中：
- 右键点击 `pom.xml`
- 选择 `Maven` -> `Reload Project`

### 第三步：启动应用

#### 方式一：IDE 中启动（推荐）
1. 打开 `src/main/java/org/javaweb/webshopbackend/WebShopBackendApplication.java`
2. 点击类旁边的绿色运行按钮
3. 或右键选择 `Run 'WebShopBackendApplication'`

#### 方式二：命令行启动
```bash
mvn spring-boot:run
```

#### 方式三：打包后启动
```bash
# 打包
mvn clean package

# 启动
java -jar target/web-shop-backend-0.0.1-SNAPSHOT.jar
```

### 第四步：验证启动

## 常见问题排查

### 1. 启动失败：端口被占用

**错误信息**：
```
Web server failed to start. Port 8080 was already in use.
```

**解决方案**：
- 方案 A：关闭占用 8080 端口的程序
- 方案 B：修改 `application.properties` 中的端口：
  ```properties
  server.port=8081
  ```

### 2. 启动失败：数据库连接失败

**错误信息**：
```
java.sql.SQLException: Access denied for user 'root'@'localhost'
```

**解决方案**：
1. 检查 MySQL 服务是否启动
2. 检查 `application.properties` 中的数据库密码是否正确
3. 检查数据库 `web_shop` 是否存在

### 3. 启动失败：Maven 依赖下载失败

**错误信息**：
```
Could not resolve dependencies
```

**解决方案**：
1. 检查网络连接
2. 配置 Maven 镜像（阿里云镜像）
3. 清理 Maven 本地仓库：`mvn clean`
4. 重新下载依赖：`mvn install`

### 4. IDEA 无法识别 Lombok

**现象**：
- `@Data`、`@Slf4j` 等注解报错
- Getter/Setter 方法找不到

**解决方案**：
1. 安装 Lombok 插件：
   - `File` -> `Settings` -> `Plugins`
   - 搜索 "Lombok"
   - 安装并重启 IDEA

2. 启用注解处理：
   - `File` -> `Settings` -> `Build, Execution, Deployment` -> `Compiler` -> `Annotation Processors`
   - 勾选 `Enable annotation processing`

### 5. API 文档无法访问

**现象**：
- 访问 http://localhost:8080/doc.html 返回 404

**解决方案**：
1. 检查 `application.properties` 中的配置：
   ```properties
   springdoc.api-docs.enabled=true
   knife4j.enable=true
   ```

2. 确认 Knife4j 依赖已正确添加到 `pom.xml`

3. 清理并重新启动项目

## 开发建议

### 1. 开发工具推荐

- **IntelliJ IDEA**: 最强大的 Java IDE
- **Postman/Apifox**: API 测试工具
- **Navicat/DataGrip**: 数据库管理工具
- **Git**: 版本控制工具

### 2. IDEA 插件推荐

- **Lombok**: 简化 Java 代码
- **MyBatisX**: MyBatis 增强插件
- **RestfulTool**: 快速定位 Controller 接口
- **Translation**: 翻译插件
- **Rainbow Brackets**: 彩虹括号

### 3. 开发流程建议

1. **需求分析** -> 确定接口功能
2. **数据库设计** -> 设计表结构
3. **实体类创建** -> 创建 Entity/VO/DTO
4. **Mapper 层** -> 编写数据访问接口
5. **Service 层** -> 编写业务逻辑
6. **Controller 层** -> 编写 API 接口
7. **测试** -> 使用 Knife4j 测试接口
8. **联调** -> 与前端联调

### 4. 代码规范建议

- 使用 Lombok 简化代码（`@Data`、`@Slf4j`）
- 使用统一响应格式（`Result`）
- 使用 Knife4j 注解编写 API 文档（`@Tag`、`@Operation`）
- 使用 `@Valid` 进行参数校验
- 异常统一由 `GlobalExceptionHandler` 处理
- 日志使用 `@Slf4j` 注解，不要使用 `System.out.println()`

## 下一步开发

现在后端配置已完成，可以开始业务开发：

### 1. 创建数据库表

编辑 `src/main/resources/database_init.sql`，创建业务表：
- 用户表（user）
- 商品表（product）
- 分类表（category）
- 订单表（order）
- 订单详情表（order_item）
- 等等...

### 2. 创建实体类

在 `pojo` 包下创建实体类，例如：
```java
@Data
@TableName("user")
public class User implements Serializable {
    @TableId(type = IdType.AUTO)
    private Long id;
    private String username;
    private String password;
    // ...
}
```

### 3. 创建 Mapper 接口

在 `mapper` 包下创建 Mapper 接口：
```java
@Mapper
public interface UserMapper extends BaseMapper<User> {
    // 自定义 SQL 方法
}
```

### 4. 创建 Service 层

在 `service` 包下创建 Service 接口和实现类：
```java
public interface UserService extends IService<User> {
    // 业务方法
}
```

### 5. 创建 Controller 层

在 `controller` 包下创建 Controller：
```java
@RestController
@RequestMapping("/api/user")
@Tag(name = "用户管理")
public class UserController {
    // API 接口
}
```
