## 🔴 一、严重业务逻辑漏洞

### 1.1 【严重】订单创建时立即扣减库存，未支付订单占用库存

**问题位置**: `OrdersServiceImpl.createOrder()`

#### 问题描述
```java
// ❌ 当前实现
@Override
@Transactional
public Orders createOrder(...) {
    // 1. 检查库存
    if (!productService.checkStock(product.getId(), cartItem.getQuantity())) {
        throw new IllegalArgumentException("库存不足");
    }
    
    // 2. 创建订单（状态=0，待付款）
    order.setStatus(0);
    this.save(order);
    
    // 3. 立即扣减库存 ❌ 问题：用户可能不支付
    productService.updateProductStock(product.getId(), -cartItem.getQuantity());
}
```

**问题分析**：
- 订单创建时（状态=0，待付款）就扣减了库存
- 如果用户不支付，库存被占用，其他用户无法购买
- 没有订单超时自动取消机制，库存可能长期被占用

#### 业务影响
```
场景1：商品库存=10
- 用户A创建订单（未支付）：库存=9
- 用户B创建订单（未支付）：库存=8
- 用户C创建订单（未支付）：库存=7
- ... 10个用户都创建订单但不支付
- 结果：库存=0，但实际没有卖出任何商品！
```

#### 修复方案

**方案1：支付时扣减库存（推荐）**
```java
@Override
@Transactional
public Orders createOrder(...) {
    // 1. 检查库存（只检查，不扣减）
    for (ShoppingCart cartItem : cartItems) {
        Product product = productService.getById(cartItem.getProductId());
        if (!productService.checkStock(product.getId(), cartItem.getQuantity())) {
            throw new IllegalArgumentException("商品库存不足");
        }
    }
    
    // 2. 创建订单（状态=0，待付款）
    order.setStatus(0);
    this.save(order);
    
    // ✅ 不扣减库存，等支付时再扣减
}

@Override
@Transactional
public void payOrder(String orderNo, Long userId, Integer paymentMethod) {
    Orders order = getOrderByNo(orderNo);
    
    // ✅ 支付时再检查并扣减库存
    List<OrderItem> orderItems = orderItemService.getByOrderId(order.getId());
    for (OrderItem item : orderItems) {
        Product product = productService.getById(item.getProductId());
        
        // 使用悲观锁检查库存
        Product productLocked = productMapper.selectByIdForUpdate(item.getProductId());
        if (productLocked.getStock() < item.getQuantity()) {
            throw new IllegalArgumentException("商品库存不足，无法支付");
        }
        
        // 扣减库存
        productService.updateProductStock(item.getProductId(), -item.getQuantity());
    }
    
    // 更新订单状态
    order.setStatus(1);
    order.setPayTime(LocalDateTime.now());
    this.updateById(order);
}
```

**方案2：预留库存机制（复杂但更优）**
```java
// 创建订单时预留库存（不实际扣减）
// 设置订单超时时间（如30分钟）
// 超时后自动释放预留库存
// 支付时从预留转为实际扣减
```

### 1.2 【严重】订单取消时库存恢复逻辑错误

**问题位置**: `OrdersServiceImpl.cancelOrder()`

#### 问题描述
```java
// ❌ 当前实现
@Override
@Transactional
public void cancelOrder(String orderNo, Long userId, String reason) {
    Orders order = getOrderByNo(orderNo);
    
    // 验证订单状态（只有待付款和待发货可以取消）
    if (order.getStatus() != 0 && order.getStatus() != 1) {
        throw new IllegalArgumentException("当前订单状态不允许取消");
    }
    
    // ❌ 问题：无论订单是否已支付，都恢复库存
    List<OrderItem> orderItems = orderItemService.getByOrderId(order.getId());
    for (OrderItem item : orderItems) {
        productService.updateProductStock(item.getProductId(), item.getQuantity());
    }
    
    order.setStatus(4);  // 已取消
    this.updateById(order);
}
```

**问题分析**：
- 订单状态=0（待付款）：创建时已扣减库存，取消时恢复 ✅ 正确
- 订单状态=1（待发货）：已支付，但创建时已扣减库存，取消时又恢复一次 ❌ **重复恢复库存！**

#### 业务影响
```
场景：
1. 创建订单（状态=0）：库存 100 → 97（扣减3）
2. 支付订单（状态=1）：库存不变，仍为97
3. 取消订单：库存 97 → 100（恢复3）

问题：如果订单创建时没有扣减库存（方案1），那么取消时恢复库存就是错误的！
```

#### 修复方案

```java
@Override
@Transactional
public void cancelOrder(String orderNo, Long userId, String reason) {
    Orders order = getOrderByNo(orderNo);
    
    // 验证订单状态
    if (order.getStatus() != 0 && order.getStatus() != 1) {
        throw new IllegalArgumentException("当前订单状态不允许取消");
    }
    
    // ✅ 根据订单状态决定是否恢复库存
    if (order.getStatus() == 0) {
        // 待付款订单：如果创建时扣减了库存，需要恢复
        // 如果创建时没扣减（方案1），则不需要恢复
        // 这里假设采用方案1（支付时扣减），所以待付款订单不需要恢复库存
        // 如果采用原方案（创建时扣减），则需要恢复：
        // List<OrderItem> orderItems = orderItemService.getByOrderId(order.getId());
        // for (OrderItem item : orderItems) {
        //     productService.updateProductStock(item.getProductId(), item.getQuantity());
        // }
    } else if (order.getStatus() == 1) {
        // 已支付订单：需要恢复库存，并处理退款
        List<OrderItem> orderItems = orderItemService.getByOrderId(order.getId());
        for (OrderItem item : orderItems) {
            productService.updateProductStock(item.getProductId(), item.getQuantity());
        }
        
        // ✅ 已支付的订单取消，需要退款
        // 这里应该调用退款服务，或者直接标记为"已取消（已退款）"
        // order.setStatus(5); // 已取消（已退款）
    }
    
    order.setStatus(4);  // 已取消
    order.setCancelTime(LocalDateTime.now());
    order.setCancelReason(reason);
    this.updateById(order);
}
```

### 1.3 【严重】退款时没有恢复库存

**问题位置**: `RefundServiceImpl.reviewRefund()`

#### 问题描述
```java
// ❌ 当前实现
@Override
@Transactional
public void reviewRefund(Long refundId, Integer status) {
    Refund refund = this.getById(refundId);
    Orders order = ordersService.getById(refund.getOrderId());
    
    if (status == 1) {
        // 同意退款
        order.setStatus(7);  // 已退款
        // ❌ 没有恢复库存！
    } else {
        // 拒绝退款
        order.setStatus(...);  // 恢复原状态
    }
    
    ordersService.updateById(order);
}
```

**问题分析**：
- 退款成功时，商品应该退回库存
- 当前实现没有恢复库存，导致库存永久丢失

#### 业务影响
```
场景：
1. 用户购买商品：库存 100 → 97
2. 用户申请退款
3. 商家同意退款：库存仍为 97 ❌ 应该是 100

结果：商品已退回，但库存没有恢复，无法再次销售
```

#### 修复方案

```java
@Override
@Transactional
public void reviewRefund(Long refundId, Integer status) {
    Refund refund = this.getById(refundId);
    Orders order = ordersService.getById(refund.getOrderId());
    
    if (status == 1) {
        // ✅ 同意退款：恢复库存
        List<OrderItem> orderItems = orderItemService.getByOrderId(order.getId());
        for (OrderItem item : orderItems) {
            productService.updateProductStock(item.getProductId(), item.getQuantity());
        }
        
        // ✅ 减少商品销量（因为退款了）
        for (OrderItem item : orderItems) {
            productService.updateProductSales(item.getProductId(), -item.getQuantity());
        }
        
        order.setStatus(7);  // 已退款
        log.info("退款申请已同意，库存已恢复");
    } else {
        // 拒绝退款
        if (order.getShipTime() != null) {
            order.setStatus(2);  // 已发货
        } else {
            order.setStatus(1);  // 待发货
        }
    }
    
    ordersService.updateById(order);
}
```

### 1.4 【严重】订单创建和库存扣减之间存在时间差，价格可能已变化

**问题位置**: `OrdersServiceImpl.createOrder()`

#### 问题描述
```java
// ❌ 当前实现
@Override
@Transactional
public Orders createOrder(...) {
    // 第1次查询商品（用于计算金额）
    BigDecimal totalAmount = BigDecimal.ZERO;
    for (ShoppingCart cartItem : cartItems) {
        Product product = productService.getById(cartItem.getProductId());
        BigDecimal itemTotal = product.getPrice().multiply(new BigDecimal(cartItem.getQuantity()));
        totalAmount = totalAmount.add(itemTotal);
    }
    
    // 创建订单
    order.setTotalAmount(totalAmount);
    this.save(order);
    
    // 第2次查询商品（用于创建订单项和扣减库存）
    for (ShoppingCart cartItem : cartItems) {
        Product product = productService.getById(cartItem.getProductId());  // ❌ 可能价格已变化
        orderItem.setUnitPrice(product.getPrice());  // ❌ 价格不一致
        productService.updateProductStock(product.getId(), -cartItem.getQuantity());
    }
}
```

**问题分析**：
- 虽然使用了 `@Transactional`，但在同一事务内两次查询商品，如果商品价格在事务期间被修改，可能导致：
  1. 订单总金额和订单项单价不一致
  2. 订单项保存的价格与计算总金额时的价格不一致

#### 修复方案

```java
@Override
@Transactional
public Orders createOrder(...) {
    // ✅ 一次性查询所有商品并保存快照
    Map<Long, Product> productMap = new HashMap<>();
    BigDecimal totalAmount = BigDecimal.ZERO;
    
    for (ShoppingCart cartItem : cartItems) {
        Product product = productService.getById(cartItem.getProductId());
        if (product == null || product.getStatus() == 0) {
            throw new IllegalArgumentException("商品不存在或已下架");
        }
        if (!productService.checkStock(product.getId(), cartItem.getQuantity())) {
            throw new IllegalArgumentException("商品库存不足");
        }
        
        // ✅ 保存商品快照（使用当前价格）
        productMap.put(product.getId(), product);
        
        BigDecimal itemTotal = product.getPrice().multiply(new BigDecimal(cartItem.getQuantity()));
        totalAmount = totalAmount.add(itemTotal);
    }
    
    // 创建订单
    order.setTotalAmount(totalAmount);
    order.setPayAmount(totalAmount);
    this.save(order);
    
    // ✅ 使用保存的商品快照创建订单项
    for (ShoppingCart cartItem : cartItems) {
        Product product = productMap.get(cartItem.getProductId());  // ✅ 使用快照
        
        OrderItem orderItem = new OrderItem();
        orderItem.setOrderId(order.getId());
        orderItem.setProductId(product.getId());
        orderItem.setProductName(product.getName());
        orderItem.setProductImage(product.getCoverImage());
        orderItem.setUnitPrice(product.getPrice());  // ✅ 价格一致
        orderItem.setQuantity(cartItem.getQuantity());
        orderItem.setTotalPrice(product.getPrice().multiply(new BigDecimal(cartItem.getQuantity())));
        orderItemService.save(orderItem);
        
        // 扣减库存
        productService.updateProductStock(product.getId(), -cartItem.getQuantity());
    }
}
```

### 1.5 【严重】订单超时未支付没有自动取消机制

**问题位置**: 全局缺失

#### 问题描述
- 用户创建订单后，如果30分钟内未支付，订单应该自动取消
- 当前系统没有定时任务处理超时订单
- 导致大量"僵尸订单"占用库存

#### 修复方案

```java
// 1. 创建定时任务
@Component
@Slf4j
public class OrderTimeoutTask {
    
    @Autowired
    private OrdersService ordersService;
    
    /**
     * 每分钟检查一次超时订单
     */
    @Scheduled(fixedRate = 60000)  // 60秒
    public void cancelTimeoutOrders() {
        log.info("开始检查超时订单");
        
        // 查询30分钟前创建的待付款订单
        LocalDateTime timeout = LocalDateTime.now().minusMinutes(30);
        
        LambdaQueryWrapper<Orders> wrapper = new LambdaQueryWrapper<>();
        wrapper.eq(Orders::getStatus, 0)  // 待付款
               .lt(Orders::getCreatedTime, timeout);
        
        List<Orders> timeoutOrders = ordersService.list(wrapper);
        
        for (Orders order : timeoutOrders) {
            try {
                // 自动取消订单
                ordersService.cancelOrder(
                    order.getOrderNo(), 
                    order.getUserId(), 
                    "订单超时未支付，自动取消"
                );
                log.info("订单超时自动取消：orderNo={}", order.getOrderNo());
            } catch (Exception e) {
                log.error("取消超时订单失败：orderNo={}", order.getOrderNo(), e);
            }
        }
        
        log.info("超时订单检查完成，共取消{}个订单", timeoutOrders.size());
    }
}

// 2. 启用定时任务
@SpringBootApplication
@EnableScheduling  // ✅ 添加此注解
public class WebShopBackendApplication {
    public static void main(String[] args) {
        SpringApplication.run(WebShopBackendApplication.class, args);
    }
}
```

### 2.2 【重要】订单金额计算可能存在精度问题

**问题位置**: `OrdersServiceImpl.createOrder()`

#### 问题描述
```java
// ❌ 当前实现
BigDecimal itemTotal = product.getPrice().multiply(new BigDecimal(cartItem.getQuantity()));
totalAmount = totalAmount.add(itemTotal);
```

**问题分析**：
- 虽然使用了 `BigDecimal`，但没有设置精度和舍入模式
- 如果商品价格是 `99.99`，数量是 `3`，结果应该是 `299.97`
- 但如果数据库存储精度不够，可能导致精度丢失

#### 修复方案

```java
// ✅ 设置精度和舍入模式
BigDecimal itemTotal = product.getPrice()
    .multiply(new BigDecimal(cartItem.getQuantity()))
    .setScale(2, RoundingMode.HALF_UP);  // 保留2位小数，四舍五入

totalAmount = totalAmount.add(itemTotal)
    .setScale(2, RoundingMode.HALF_UP);
```

### 2.4 【重要】退款金额验证不完整

**问题位置**: `RefundServiceImpl.applyRefund()`

#### 问题描述
```java
// ❌ 当前实现
```

**问题分析**：

```java
@Override
@Transactional
public void applyRefund(Refund refund) {
    Orders order = ordersService.getById(refund.getOrderId());
    
    // ✅ 验证退款金额（应该使用实付金额）
    if (refund.getRefundAmount().compareTo(order.getPayAmount()) > 0) {
        throw new IllegalArgumentException("退款金额不能超过实付金额");
    }
    if (refund.getRefundAmount().compareTo(BigDecimal.ZERO) <= 0) {
        throw new IllegalArgumentException("退款金额必须大于0");
    }
    
    // ✅ 部分退款：验证是否已有部分退款
    LambdaQueryWrapper<Refund> wrapper = new LambdaQueryWrapper<>();

    List<Refund> existingRefunds = this.list(wrapper);
    BigDecimal totalRefunded = existingRefunds.stream()
        .map(Refund::getRefundAmount)
        .reduce(BigDecimal.ZERO, BigDecimal::add);
    
    BigDecimal remainingAmount = order.getPayAmount().subtract(totalRefunded);

        throw new IllegalArgumentException(
            String.format("退款金额超过可退金额，剩余可退：%.2f元", remainingAmount)
```

### 2.5 【重要】订单状态流转不完整

**问题位置**: 全局订单状态管理

#### 问题描述
当前订单状态定义：
- 0: 待付款
- 1: 待发货（已支付）
- 2: 已发货
- 3: 已收货
- 4: 已取消
- 6: 退款中
- 7: 已退款

**问题分析**：
- 缺少"部分退款"状态
- 缺少"已取消（已退款）"状态
- 状态流转规则不清晰

#### 修复方案

```java
// 定义订单状态枚举
public enum OrderStatus {
    UNPAID(0, "待付款"),
    PAID(1, "待发货"),
    SHIPPED(2, "已发货"),
    RECEIVED(3, "已收货"),
    CANCELLED(4, "已取消"),
    CANCELLED_REFUNDED(5, "已取消（已退款）"),
    REFUNDING(6, "退款中"),
    REFUNDED(7, "已退款"),
    PARTIALLY_REFUNDED(8, "部分退款");
    
    private final Integer code;
    private final String desc;
}

// 定义状态流转规则
public class OrderStatusFlow {
    private static final Map<Integer, List<Integer>> ALLOWED_TRANSITIONS = new HashMap<>();
    
    static {
        ALLOWED_TRANSITIONS.put(0, Arrays.asList(1, 4));  // 待付款 → 待发货/已取消
        ALLOWED_TRANSITIONS.put(1, Arrays.asList(2, 4, 6));  // 待发货 → 已发货/已取消/退款中
        ALLOWED_TRANSITIONS.put(2, Arrays.asList(3, 6));  // 已发货 → 已收货/退款中
        ALLOWED_TRANSITIONS.put(3, Arrays.asList(6));  // 已收货 → 退款中
        ALLOWED_TRANSITIONS.put(6, Arrays.asList(1, 2, 3, 7, 8));  // 退款中 → 恢复原状态/已退款/部分退款
    }
    
    public static boolean canTransition(Integer fromStatus, Integer toStatus) {
        List<Integer> allowed = ALLOWED_TRANSITIONS.get(fromStatus);
        return allowed != null && allowed.contains(toStatus);
    }
}

// 在状态变更时验证
public void updateOrderStatus(Long orderId, Integer newStatus) {
    Orders order = this.getById(orderId);
    
    if (!OrderStatusFlow.canTransition(order.getStatus(), newStatus)) {
        throw new IllegalArgumentException(
            String.format("订单状态不能从 %d 变更为 %d", order.getStatus(), newStatus)
        );
    }
    
    order.setStatus(newStatus);
    this.updateById(order);
}
```

### 3.4 【一般】确认收货后没有自动完成时间限制

**问题位置**: `OrdersServiceImpl.confirmReceive()`

#### 问题描述
- 已发货订单，用户可以在任意时间确认收货
- 没有"自动确认收货"机制（如发货后7天自动确认）

#### 修复方案

```java
// 添加定时任务：自动确认收货
@Scheduled(fixedRate = 86400000)  // 每天执行一次
public void autoConfirmReceive() {
    // 查询7天前发货但未确认收货的订单
    LocalDateTime deadline = LocalDateTime.now().minusDays(7);
    
    LambdaQueryWrapper<Orders> wrapper = new LambdaQueryWrapper<>();
    wrapper.eq(Orders::getStatus, 2)  // 已发货
           .lt(Orders::getShipTime, deadline);
    
    List<Orders> orders = ordersService.list(wrapper);
    
    for (Orders order : orders) {
        try {
            ordersService.confirmReceive(order.getOrderNo(), order.getUserId());
            log.info("订单自动确认收货：orderNo={}", order.getOrderNo());
        } catch (Exception e) {
            log.error("自动确认收货失败：orderNo={}", order.getOrderNo(), e);
        }
    }
}
```
