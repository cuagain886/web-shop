# WebShop API 安全漏洞清单（简要版）

生成时间：2025-11-06

---

## 🔴 严重漏洞（必须立即修复）

### 1. 缺少身份认证机制
- **影响**: 所有API都可以匿名访问
- **风险**: 任何人都可以调用所有接口
- **修复**: 引入JWT认证

### 2. 缺少权限控制
- **影响**: 普通用户可以执行管理员操作
- **风险**: 可以禁用用户、删除商品、审核退款等
- **修复**: 实现RBAC权限模型

### 3. 越权访问漏洞
- **影响**: 以下接口存在问题
  ```
  ❌ UserController - 修改其他用户信息/密码
  ❌ UserAddressController - 查看/修改其他用户地址
  ❌ OrdersController - 查看/取消其他用户订单
  ❌ ShoppingCartController - 操作其他用户购物车
  ❌ UserFavoriteController - 操作其他用户收藏
  ❌ BrowsingHistoryController - 查看其他用户浏览历史
  ❌ RefundController - 取消其他用户退款
  ❌ ProductReviewController - 删除其他用户评价
  ```
- **风险**: 用户A可以操作用户B的所有数据
- **修复**: 添加userId验证

### 4. 库存并发扣减漏洞
- **位置**: `OrdersServiceImpl.createOrder()`
- **风险**: 高并发下可能超卖（库存变负数）
- **修复**: 使用悲观锁或乐观锁

### 5. 支付金额可被篡改
- **位置**: `OrdersServiceImpl.payOrder()`
- **风险**: 可以修改订单金额后再支付
- **修复**: 对接真实支付API并验证金额

## 🟠 重要漏洞（高优先级修复）

### 6. 密码修改缺少二次验证
- **位置**: `UserService.changePassword()`
- **修复**: 添加短信验证码

### 7. 评价可以重复提交
- **位置**: `ProductReviewService.addReview()`
- **修复**: 检查是否已评价

### 8. 退款可以重复申请
- **位置**: `RefundService.applyRefund()`
- **修复**: 检查是否已有退款申请

### 9. 商品分类删除缺少检查
- **位置**: `ProductCategoryService.deleteCategory()`
- **修复**: 检查是否有子分类或商品

### 10. 购物车数量可以为负数
- **位置**: `ShoppingCartService`
- **修复**: 添加参数验证(1-999)

### 11. 订单地址可能被修改
- **位置**: `OrdersService.createOrder()`
- **修复**: 存储地址快照而非地址ID

## 🟡 其他漏洞（后续优化）

### 12-19. 其他安全问题
- 敏感信息泄露（错误信息过于详细）

- 缺少操作日志

## 📝 代码修复示例

### 修复1: 添加userId验证
```java
// UserController - 修改用户信息
@PutMapping("/{userId}")
public Result<Void> updateUser(@PathVariable Long userId, @RequestBody User user) {
    // ✅ 添加权限验证
    Long currentUserId = UserContext.getCurrentUserId();
    if (!currentUserId.equals(userId)) {
        throw new ForbiddenException("无权修改其他用户信息");
    }
    
    user.setId(userId);
    userService.updateUserInfo(user);
    return Result.success("更新成功");
}
```

```java
// ProductMapper.xml - 使用数据库原子操作
<update id="updateStock">
    UPDATE product 
    SET stock = stock + #{quantity}
    WHERE id = #{productId} 
      AND stock + #{quantity} >= 0  <!-- ✅ 防止库存为负 -->
</update>

// OrdersServiceImpl - 使用FOR UPDATE锁
@Transactional
public Orders createOrder(...) {
    for (ShoppingCart cartItem : cartItems) {
        // ✅ 加悲观锁
        Product product = productMapper.selectByIdForUpdate(cartItem.getProductId());
        
        if (product.getStock() < cartItem.getQuantity()) {
            throw new IllegalArgumentException("商品库存不足");
        }
        
        productService.updateStock(cartItem.getProductId(), -cartItem.getQuantity());
    }
}
```
