# 用户注册错误修复记录

**问题：** 测试用户注册时出现 500 错误
**错误信息：** `Invalid bound statement (not found): org.javaweb.webshopbackend.mapper.UserMapper.selectByUsernameIgnoreDeleted`

**修复日期：** 2025-11-03  
**修复状态：** ✅ 已完成

---

## 🔍 问题分析

### 错误原因

1. **缺少XML映射文件**
   - `UserMapper` 接口中定义了自定义方法 `selectByUsernameIgnoreDeleted`
   - 但没有对应的 XML 映射文件实现
   - MyBatis 无法找到绑定的 SQL 语句

2. **role 字段类型不匹配**
   - 数据库：`role TINYINT` (0-普通用户，1-商家)
   - 实体类：原先定义为 `String role`
   - 导致类型不一致

---

## 🔧 修复方案

### 修复 1: 创建 UserMapper.xml

**文件位置：** `src/main/resources/mapper/UserMapper.xml`

**创建内容：**
```xml
<?xml version="1.0" encoding="UTF-8"?>
<!DOCTYPE mapper PUBLIC "-//mybatis.org//DTD Mapper 3.0//EN" 
    "http://mybatis.org/dtd/mybatis-3-mapper.dtd">
<mapper namespace="org.javaweb.webshopbackend.mapper.UserMapper">

    <!-- 根据用户名查询用户（忽略逻辑删除） -->
    <select id="selectByUsernameIgnoreDeleted" resultType="...">
        SELECT * FROM user WHERE username = #{username} LIMIT 1
    </select>

    <!-- 根据手机号查询用户（忽略逻辑删除） -->
    <select id="selectByPhoneIgnoreDeleted" resultType="...">
        SELECT * FROM user WHERE phone = #{phone} LIMIT 1
    </select>

    <!-- 更新最后登录时间 -->
    <update id="updateLastLoginTime">
        UPDATE user SET last_login_time = NOW() WHERE id = #{userId}
    </update>

</mapper>
```

## ⚠️ 注意事项

### 前后端 role 字段不一致

**问题：**
- 后端现在使用 `Integer` (0, 1)
- 前端可能还在使用 `String` ('user', 'merchant')

**影响范围：**
- `Header.vue` - 判断用户角色
- `router/index.js` - 路由守卫
- `api/user.js` - Mock 数据

**临时解决方案：**

在前端判断时修改为：
```javascript
// 修改前
if (userStore.userInfo?.role === 'merchant') { ... }

// 修改后
if (userStore.userInfo?.role === 1) { ... }
```

**长期解决方案（推荐）：**

在后端添加一个转换器，返回给前端时转换：
```java
// 在 Controller 返回前转换
public String getRoleString() {
    return this.role == 1 ? "merchant" : "user";
}
```

或者在前端统一处理：
```javascript
// 在 store 中添加计算属性
computed: {
  roleString() {
    return this.userInfo?.role === 1 ? 'merchant' : 'user'
  }
}
```

## 💡 经验总结

### 常见 MyBatis 错误原因

1. **XML 文件缺失或路径错误**
2. **namespace 配置错误**
3. **方法名不匹配**
4. **参数映射错误**
5. **resultType/resultMap 配置错误**

### 预防措施

1. **及时创建 XML 文件** - 定义自定义方法后立即创建对应 XML
2. **检查配置** - 确保 `mybatis-plus.mapper-locations` 配置正确
3. **类型一致** - 保持数据库、实体类、前端的字段类型一致
4. **单元测试** - 为 Mapper 方法编写单元测试
