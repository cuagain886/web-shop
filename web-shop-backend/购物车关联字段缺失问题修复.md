# 购物车关联字段缺失问题修复

**问题发现日期：** 2025-11-04  
**错误信息：** `There is no getter for property named 'product' in 'class org.javaweb.webshopbackend.pojo.entity.ShoppingCart'`  
**状态：** ✅ 已修复

## 🔍 问题描述

### 用户反馈

调用购物车相关 API 时出现错误：

**操作步骤：**
1. 调用 `POST /api/cart` 添加商品到购物车 ✅ 成功


---

## 🔍 问题分析

### 根本原因

**ShoppingCart 实体类中缺少 `product` 关联字段**，但在 Mapper XML 中定义了关联映射。

### 涉及的代码

#### 1. ShoppingCartMapper.xml（定义了关联）

```xml
<resultMap id="CartWithProductMap" type="org.javaweb.webshopbackend.pojo.entity.ShoppingCart">
    <!-- ... 其他字段 ... -->
    
    <!-- 关联商品信息 -->
    <association property="product" javaType="org.javaweb.webshopbackend.pojo.entity.Product">
        <id column="p_id" property="id"/>
        <result column="p_name" property="name"/>
        <result column="p_price" property="price"/>
        <!-- ... -->
    </association>
</resultMap>

<select id="selectCartWithProduct" resultMap="CartWithProductMap">
    SELECT
        sc.*,
        p.id AS p_id,
        p.name AS p_name,
        p.price AS p_price,
        -- ... --
    FROM `shopping_cart` sc
    LEFT JOIN `product` p ON sc.product_id = p.id
    WHERE sc.user_id = #{userId}
</select>
```

**问题：** XML 中定义了 `property="product"`，期望 ShoppingCart 有这个字段。

#### 2. ShoppingCart.java（缺少字段）

**修复前：**
```java
@Data
@TableName("shopping_cart")
public class ShoppingCart implements Serializable {
    private Long id;
    private Long userId;
    private Long productId;  // ← 只有商品ID
    private String specInfo;
    private Integer quantity;
    private Integer checked;
    private LocalDateTime createdTime;
    private LocalDateTime updatedTime;
    
    // ❌ 缺少 Product product 字段
}
```

**错误流程：**
1. MyBatis 执行 SQL，查询购物车和商品信息
2. 尝试通过 `<association property="product">` 映射到 ShoppingCart.product
3. ❌ 找不到 `product` 字段的 getter 方法
4. 抛出异常：`There is no getter for property named 'product'`

---

## ✅ 解决方案

### 添加 product 关联字段

在 ShoppingCart 实体类中添加 `Product` 类型的字段：

```java
@Data
@TableName("shopping_cart")
public class ShoppingCart implements Serializable {
    private Long id;
    private Long userId;
    private Long productId;
    private String specInfo;
    private Integer quantity;
    private Integer checked;
    private LocalDateTime createdTime;
    private LocalDateTime updatedTime;

    /**
     * 关联的商品信息（非数据库字段）
     */
    @TableField(exist = false)  // ← 重要！标记为非数据库字段
    private Product product;
}
```

---

## 🎯 关键知识点

### @TableField(exist = false)

**作用：** 标记该字段不是数据库表中的列，只用于业务逻辑。

**为什么需要：**
- ✅ 告诉 MyBatis-Plus：这个字段不对应数据库列
- ✅ 避免在 INSERT/UPDATE 时尝试写入该字段
- ✅ 只用于查询结果的映射

**对比：**

| 注解 | 含义 | 用途 |
|------|------|------|
| **无注解** | 数据库字段 | `productId` 等真实字段 |
| **@TableField(exist = false)** | 非数据库字段 | `product` 等关联对象 |
| **@TableField(select = false)** | 不参与查询 | 敏感字段 |

## ⚠️ 注意事项

### 1. @TableField(exist = false) 是必须的

**如果不加这个注解：**
- ❌ MyBatis-Plus 会认为 `product` 是数据库字段
- ❌ INSERT/UPDATE 时会尝试写入该字段
- ❌ 导致 SQL 错误：`Unknown column 'product' in 'field list'`

### 2. 关联字段不会自动填充

**关联字段只在使用自定义 Mapper 方法时才会有值：**

```java
// ✅ 有 product 信息（使用了 selectCartWithProduct）
List<ShoppingCart> carts = cartMapper.selectCartWithProduct(userId);

// ❌ 没有 product 信息（使用 MyBatis-Plus 默认方法）
List<ShoppingCart> carts = cartMapper.selectList(new QueryWrapper<>().eq("user_id", userId));
```

### 3. JSON 序列化

**Lombok 的 @Data 会自动生成 getter/setter：**
- ✅ `getProduct()` → JSON 返回时会包含 `product` 字段
- ✅ 如果 `product` 为 null，默认不会序列化（取决于配置）

## 🔄 举一反三

### MyBatis ResultMap 关联映射规则

| XML 定义 | Java 字段要求 | 注解要求 |
|---------|--------------|---------|
| `<association property="product">` | 必须有 `private Product product;` | `@TableField(exist = false)` |
| `<collection property="items">` | 必须有 `private List<OrderItem> items;` | `@TableField(exist = false)` |

**记忆技巧：**
- XML 中 `property="xxx"` → Java 中必须有字段 `xxx`
- 如果是关联对象 → 必须加 `@TableField(exist = false)`

## 🎯 总结

### 问题本质

**MyBatis ResultMap 与 Java 实体类不匹配**
- XML 定义了关联映射 `property="product"`
- Java 类中没有对应的 `product` 字段
- MyBatis 无法完成映射，抛出异常

### 解决方法

**在实体类中添加关联字段，并标记为非数据库字段**
```java
@TableField(exist = false)
private Product product;
```

### 预防措施

**创建 Mapper XML 时，同步检查实体类：**
1. 如果使用了 `<association>`，确保实体类有对应字段
2. 如果使用了 `<collection>`，确保实体类有对应 List 字段
3. 所有关联字段都要加 `@TableField(exist = false)`
