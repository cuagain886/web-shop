## 📊 数据库设计

### 商品表（product）图片相关字段

```sql
CREATE TABLE `product` (
    `id` bigint NOT NULL AUTO_INCREMENT COMMENT '商品ID',
    `name` varchar(200) NOT NULL COMMENT '商品名称',
    -- ... 其他字段 ...
    
    -- 图片相关字段（重点）
    `cover_image` varchar(255) DEFAULT NULL COMMENT '封面图片URL',
    `images` varchar(1000) DEFAULT NULL COMMENT '商品图片（JSON数组）',
    
    -- ... 其他字段 ...
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### 字段说明

| 字段名 | 类型 | 说明 | 用途 |
|--------|------|------|------|
| `cover_image` | varchar(255) | 封面图片URL | 商品列表展示的主图 |
| `images` | varchar(1000) | 商品图片JSON数组 | 商品详情页的轮播图 |

## 🎯 存储方案设计

### 方案1：存储图片URL（推荐，本项目采用）

**优点：**
- ✅ 数据库字段简单，存储字符串
- ✅ 图片独立存储，便于CDN加速
- ✅ 可以使用第三方对象存储（阿里云OSS、七牛云等）
- ✅ 图片更新不影响数据库

**缺点：**
- ❌ 需要维护图片存储服务
- ❌ 图片URL可能失效

**数据示例：**
```json
// 数据库存储
{
  "cover_image": "https://cdn.example.com/products/iphone15-cover.jpg",
  "images": "[\"https://cdn.example.com/products/iphone15-1.jpg\",\"https://cdn.example.com/products/iphone15-2.jpg\",\"https://cdn.example.com/products/iphone15-3.jpg\"]"
}
```

---

### 方案2：存储Base64编码（不推荐）

**优点：**
- ✅ 图片和数据一起存储，不会丢失

**缺点：**
- ❌ 数据库体积膨胀（Base64比原图大33%）
- ❌ 查询效率降低
- ❌ 无法使用CDN
- ❌ 传输流量大

**不推荐使用！**

---

### 方案3：本地文件系统（适合小项目）

**优点：**
- ✅ 无需第三方服务
- ✅ 配置简单

**缺点：**
- ❌ 集群部署困难
- ❌ 无法使用CDN
- ❌ 备份麻烦

**数据示例：**
```json
{
  "cover_image": "/static/uploads/products/2025/11/iphone15-cover.jpg",
  "images": "[\"/static/uploads/products/2025/11/iphone15-1.jpg\",\"/static/uploads/products/2025/11/iphone15-2.jpg\"]"
}
```

## 📤 图片上传实现

### 1. 后端接口（需要实现）

**文件：** `FileController.java`

```java
@RestController
@RequestMapping("/api/file")
@Tag(name = "文件管理", description = "文件上传下载接口")
public class FileController {
    
    @Value("${file.upload.path}")
    private String uploadPath;
    
    @Value("${file.base-url}")
    private String baseUrl;
    
    /**
     * 上传商品图片
     */
    @PostMapping("/upload/product")
    @Operation(summary = "上传商品图片")
    public Result<Map<String, String>> uploadProductImage(
            @RequestParam("file") MultipartFile file) {
        
        try {
            // 1. 验证文件
            if (file.isEmpty()) {
                throw new IllegalArgumentException("文件不能为空");
            }
            
            // 验证文件类型
            String contentType = file.getContentType();
            if (!contentType.startsWith("image/")) {
                throw new IllegalArgumentException("只能上传图片文件");
            }
            
            // 验证文件大小（5MB）
            long maxSize = 5 * 1024 * 1024;
            if (file.getSize() > maxSize) {
                throw new IllegalArgumentException("文件大小不能超过5MB");
            }
            
            // 2. 生成文件名（防止重复）
            String originalFilename = file.getOriginalFilename();
            String extension = originalFilename.substring(
                originalFilename.lastIndexOf("."));
            String filename = UUID.randomUUID().toString() + extension;
            
            // 3. 按日期创建目录（便于管理）
            String datePath = LocalDate.now().format(
                DateTimeFormatter.ofPattern("yyyy/MM/dd"));
            String relativePath = "products/" + datePath + "/" + filename;
            
            // 4. 保存文件
            File destFile = new File(uploadPath + "/" + relativePath);
            destFile.getParentFile().mkdirs();
            file.transferTo(destFile);
            
            // 5. 返回访问URL
            String url = baseUrl + "/" + relativePath;
            
            Map<String, String> result = new HashMap<>();
            result.put("url", url);
            result.put("filename", filename);
            
            return Result.success("上传成功", result);
            
        } catch (Exception e) {
            log.error("文件上传失败", e);
            throw new RuntimeException("文件上传失败：" + e.getMessage());
        }
    }
    
    /**
     * 批量上传商品图片
     */
    @PostMapping("/upload/product/batch")
    @Operation(summary = "批量上传商品图片")
    public Result<List<String>> uploadProductImages(
            @RequestParam("files") MultipartFile[] files) {
        
        List<String> urls = new ArrayList<>();
        
        for (MultipartFile file : files) {
            Result<Map<String, String>> result = uploadProductImage(file);
            urls.add(result.getData().get("url"));
        }
        
        return Result.success("批量上传成功", urls);
    }
}
```

### 2. 前端上传组件

```vue
<template>
  <div class="image-upload">
    <!-- 封面图上传 -->
    <el-form-item label="商品封面">
      <el-upload
        :action="uploadUrl"
        :on-success="handleCoverSuccess"
        :before-upload="beforeUpload"
        :show-file-list="false"
        accept="image/*"
      >
        <img v-if="form.coverImage" :src="form.coverImage" class="cover-preview" />
        <el-icon v-else class="upload-icon"><Plus /></el-icon>
      </el-upload>
    </el-form-item>
    
    <!-- 商品图片上传（多张） -->
    <el-form-item label="商品图片">
      <el-upload
        :action="uploadUrl"
        :on-success="handleImagesSuccess"
        :before-upload="beforeUpload"
        :file-list="imageList"
        list-type="picture-card"
        accept="image/*"
        multiple
        :limit="5"
      >
        <el-icon><Plus /></el-icon>
      </el-upload>
    </el-form-item>
  </div>
</template>

<script setup>
import { ref } from 'vue'
import { ElMessage } from 'element-plus'
import { Plus } from '@element-plus/icons-vue'

const uploadUrl = 'http://localhost:8080/api/file/upload/product'

const form = ref({
  coverImage: '',
  images: []
})

const imageList = ref([])

// 上传前验证
const beforeUpload = (file) => {
  const isImage = file.type.startsWith('image/')
  const isLt5M = file.size / 1024 / 1024 < 5
  
  if (!isImage) {
    ElMessage.error('只能上传图片文件！')
    return false
  }
  if (!isLt5M) {
    ElMessage.error('图片大小不能超过 5MB！')
    return false
  }
  return true
}

// 封面上传成功
const handleCoverSuccess = (response) => {
  if (response.code === 200) {
    form.value.coverImage = response.data.url
    ElMessage.success('封面上传成功')
  }
}

// 商品图片上传成功
const handleImagesSuccess = (response) => {
  if (response.code === 200) {
    form.value.images.push(response.data.url)
    imageList.value.push({ url: response.data.url })
    ElMessage.success('图片上传成功')
  }
}
</script>

  height: 148px;
  display: flex;
  align-items: center;
  justify-content: center;
  border: 1px dashed #d9d9d9;
}
```

## 🎨 图片存储方案对比

| 方案 | 存储位置 | 成本 | 性能 | 可靠性 | 推荐度 |
|------|----------|------|------|--------|--------|
| **本地文件系统** | 项目服务器 | 低 | 一般 | 低 | ⭐⭐ |
| **对象存储（OSS）** | 阿里云/七牛云 | 中 | 高 | 高 | ⭐⭐⭐⭐⭐ |
| **CDN加速** | 全球节点 | 高 | 极高 | 极高 | ⭐⭐⭐⭐⭐ |
| **Base64编码** | 数据库 | 高 | 极低 | 高 | ❌ 不推荐 |

```javascript
  const options = {

    maxWidthOrHeight: 1920, // 最大宽高

  }
  
  try {
    const compressedFile = await imageCompression(file, options)

  }
```
