# WebShop 后端配置说明

## 技术栈

- **Spring Boot 3.5.7**
- **MyBatis-Plus 3.5.7**
- **MySQL 8.0+**
- **Lombok**
- **Druid 数据库连接池**
- **Knife4j API 文档工具**

## 配置类说明

### 1. MyBatisPlusConfig
**位置**: `config/MyBatisPlusConfig.java`

**功能**:
- 配置 MyBatis-Plus 拦截器
- 分页插件（支持 MySQL 分页查询）
- 乐观锁插件（防止并发更新冲突）
- 防止全表更新删除插件（防止误操作）

**核心配置**:
```java
@Configuration
@EnableTransactionManagement
@MapperScan("org.javaweb.webshopbackend.mapper")
public class MyBatisPlusConfig {
    @Bean
    public MybatisPlusInterceptor mybatisPlusInterceptor() {
        // 分页、乐观锁、防全表更新删除
    }
}
```

### 2. CorsConfig
**位置**: `config/CorsConfig.java`

**功能**:
- 解决前后端分离时的跨域问题
- 允许前端（http://localhost:5174）访问后端 API

**核心配置**:
- 允许所有域名跨域（生产环境建议限制）
- 允许所有请求方法（GET、POST、PUT、DELETE）
- 允许携带 Cookie
- 预检请求缓存 1 小时

### 3. WebMvcConfig
**位置**: `config/WebMvcConfig.java`

**功能**:
- 配置静态资源映射
- 配置 JSON 序列化/反序列化规则
- 配置上传文件访问路径

**核心配置**:
- 静态资源映射：`/uploads/**` -> `file:uploads/`
- JSON 日期格式：`yyyy-MM-dd HH:mm:ss`
- 时区设置：`GMT+8`

### 4. DruidConfig
**位置**: `config/DruidConfig.java`

**功能**:
- 配置 Druid 数据库连接池监控
- 提供数据库连接池状态监控页面

**访问地址**:
- 监控页面：http://localhost:8080/druid/
- 无需登录即可访问（可配置账号密码）

**监控内容**:
- 数据源信息
- SQL 监控
- URI 监控
- Session 监控

### 5. GlobalExceptionHandler
**位置**: `config/GlobalExceptionHandler.java`

**功能**:
- 全局异常统一处理
- 返回统一的错误响应格式

**处理的异常类型**:
- `MethodArgumentNotValidException` - 参数校验失败（@Valid）
- `BindException` - 参数绑定失败
- `IllegalArgumentException` - 非法参数
- `NullPointerException` - 空指针异常
- `RuntimeException` - 运行时异常
- `Exception` - 其他所有异常

**错误响应格式**:
```json
{
  "code": 400,
  "message": "参数校验失败",
  "errors": {
    "username": "用户名不能为空",
    "password": "密码长度必须在6-20位之间"
  }
}
```

### 6. Knife4jConfig
**位置**: `config/Knife4jConfig.java`

**功能**:
- 配置 API 文档工具
- 提供在线 API 文档和测试界面

**访问地址**:
- API 文档：http://localhost:8080/doc.html
- OpenAPI JSON：http://localhost:8080/v3/api-docs

**文档信息**:
- 标题：WebShop 电商平台 API 文档
- 版本：1.0.0
- 语言：中文

## 通用类说明

### 1. Result<T>
**位置**: `pojo/common/Result.java`

**功能**: 统一响应结果包装类

**字段**:
- `code`: 响应码（200-成功，其他-失败）
- `message`: 响应消息
- `data`: 响应数据

**使用示例**:
```java
// 成功响应
return Result.success(user);
return Result.success("登录成功", token);

// 失败响应
return Result.error("用户名或密码错误");
return Result.error(401, "未授权");
```

**响应格式**:
```json
{
  "code": 200,
  "message": "操作成功",
  "data": {
    "id": 1,
    "username": "admin"
  }
}
```

### 2. PageResult<T>
**位置**: `pojo/common/PageResult.java`

**功能**: 分页响应结果包装类

**字段**:
- `records`: 数据列表
- `total`: 总记录数
- `current`: 当前页码
- `size`: 每页大小
- `pages`: 总页数

**使用示例**:
```java
Page<User> page = userService.page(new Page<>(current, size));
PageResult<User> result = new PageResult<>(
    page.getRecords(),
    page.getTotal(),
    page.getCurrent(),
    page.getSize()
);
return Result.success(result);
```

## 启动步骤

### 1. 配置数据库
修改 `application.properties` 中的数据库密码：
```properties
spring.datasource.password=你的数据库密码
```

### 2. 创建数据库
执行 `resources/database_init.sql` 创建数据库和表：
```sql
CREATE DATABASE IF NOT EXISTS web_shop CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
USE web_shop;
```

### 3. 启动应用
运行主类 `WebShopBackendApplication.java`

### 4. 验证配置

访问以下地址验证配置是否成功：

1. **API 文档**：http://localhost:8080/doc.html
   - 查看所有 API 接口
   - 在线测试接口

2. **Druid 监控**：http://localhost:8080/druid/
   - 查看数据库连接池状态
   - 监控 SQL 执行情况

3. **健康检查**：http://localhost:8080/test/hello
   - 测试接口是否正常响应

## 常见问题

### 1. 跨域问题
**现象**: 前端请求被浏览器拦截，提示 CORS 错误

**解决**: 已配置 `CorsConfig`，允许所有域名跨域访问

### 2. 分页查询不生效
**现象**: 查询返回所有数据，未分页

**解决**: 确保使用 MyBatis-Plus 的 `Page` 对象：
```java
Page<User> page = new Page<>(current, size);
userMapper.selectPage(page, queryWrapper);
```

### 3. JSON 时间格式问题
**现象**: 时间字段返回时间戳而非格式化字符串

**解决**: 已在 `WebMvcConfig` 中配置统一格式 `yyyy-MM-dd HH:mm:ss`

### 4. 逻辑删除不生效
**现象**: 删除操作直接删除数据库记录

**解决**: 
- 确保实体类有 `@TableLogic` 注解
- 确保 `application.properties` 配置了逻辑删除字段
