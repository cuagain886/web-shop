# 数据库表结构优化说明

## 📋 优化概览

**优化时间**: 2025-11-03  
**优化目标**: 适配前端功能，解决关键问题  
**优化方案**: 方案 A（最小调整）

---

## ✅ 主要改动（10项）

### 1. 合并用户表 ⭐⭐⭐ 最重要

**改动前**:
```sql
CREATE TABLE `user` (...)    -- 普通用户
CREATE TABLE `admin` (...)   -- 管理员（独立表）
```

**改动后**:
```sql
CREATE TABLE `user` (
  ...
  `role` varchar(20) NOT NULL DEFAULT 'user' COMMENT '角色（user-普通用户，merchant-商家）',
  ...
)
```

**原因**: 
- 前端用 `role` 字段区分用户类型
- 避免登录时查询两张表
- 简化用户管理逻辑

**影响**: 
- ✅ 与前端完美适配
- ✅ 登录逻辑统一
- ⚠️ 需要在 Entity 类中处理角色映射

---

### 2. 订单表改名 ⭐⭐⭐

**改动**:
```sql
-- 改前
CREATE TABLE `order` (...)

-- 改后
CREATE TABLE `orders` (...)
```

**原因**: `order` 是 MySQL 关键字，避免冲突

**影响**: 
- ✅ 不再需要每次都用反引号
- ⚠️ SQL 语句和代码中都要用 `orders`

---

### 3. 添加逻辑删除字段 ⭐⭐

**改动**: 所有核心业务表添加
```sql
`deleted` tinyint NOT NULL DEFAULT 0 COMMENT '逻辑删除（0-未删除，1-已删除）'
```

**涉及表**:
- user
- user_address
- product_category
- product
- orders
- product_review
- refund

**原因**: MyBatis-Plus 配置了逻辑删除，必须有字段支持

**影响**:
- ✅ 数据不会真正删除
- ✅ 符合业务规范
- ✅ 支持数据恢复

---

### 4. 添加物流字段 ⭐⭐

**改动**: `orders` 表添加
```sql
`express_company` varchar(50) DEFAULT NULL COMMENT '物流公司',
`tracking_no` varchar(100) DEFAULT NULL COMMENT '物流单号',
```

**原因**: 
- 前端订单详情页需要显示物流信息
- 管家发货时需要填写物流单号

**影响**: ✅ 支持物流跟踪功能

---

### 5. 新增商品评价表 ⭐⭐⭐

**新表**: `product_review`

**字段**:
```sql
- order_item_id (关联订单项，一个订单项只能评价一次)
- user_id
- product_id
- rating (1-5星)
- content (评价内容)
- images (评价图片JSON)
- reply (商家回复)
- is_reviewed (是否已评价)
```

**前端功能**:
- 用户个人中心 -> 我的评价
- 待评价列表
- 已评价列表
- 评价弹窗

---

### 6. 新增用户收藏表 ⭐⭐

**新表**: `user_favorite`

**字段**:
```sql
- user_id
- product_id
- created_time (收藏时间)
- UNIQUE KEY (user_id, product_id) -- 同一用户对同一商品只能收藏一次
```

**前端功能**:
- 用户个人中心 -> 我的收藏
- 收藏商品列表
- 取消收藏
- 清空收藏

---

### 7. 新增浏览历史表 ⭐⭐

**新表**: `browsing_history`

**字段**:
```sql
- user_id
- product_id
- view_time (浏览时间)
```

**前端功能**:
- 用户个人中心 -> 浏览历史
- 历史记录列表
- 删除记录
- 清空历史

**特点**: 不使用逻辑删除，直接物理删除

---

### 8. 新增操作日志表 ⭐⭐⭐

**新表**: `operation_log`

**字段**:
```sql
- operator_id (操作人ID)
- operator_name (操作人名称)
- operation_type (操作类型：商品管理、订单管理等)
- operation_content (操作内容)
- operation_object (操作对象：商品ID、订单号等)
- ip_address (IP地址)
- created_time (操作时间)
```

**前端功能**:
- 管理端 -> 操作日志
- 日志列表展示
- 分类筛选
- 关键词搜索
- 时间范围查询

---

### 9. 优化商品表 ⭐

**新增字段**:
```sql
`is_hot` tinyint NOT NULL DEFAULT 0 COMMENT '是否热销（0-否，1-是）',
`is_recommend` tinyint NOT NULL DEFAULT 0 COMMENT '是否推荐（0-否，1-是）',
`images` varchar(1000) DEFAULT NULL COMMENT '商品图片（JSON数组）',
`specs` text COMMENT '商品规格（JSON格式）',
```

**原因**:
- 前端首页有热销商品、推荐商品区
- 商品详情需要多图展示
- 规格用 JSON 存储更灵活

**JSON 格式示例**:
```json
// images
["img1.jpg", "img2.jpg", "img3.jpg"]

// specs
{
  "颜色": ["红色", "蓝色", "黑色"],
  "尺寸": ["S", "M", "L", "XL"]
}
```

## ⚠️ 注意事项

### 1. 执行顺序
必须按顺序执行：
1. 先删除旧数据库（如果存在）
2. 执行新的 `database_init.sql`
3. 验证表结构
4. 测试功能

### 2. 数据迁移
如果已有旧数据：
- `admin` 表数据需要迁移到 `user` 表
- 添加 `role='merchant'` 标识
- `order` 表改名为 `orders`

### 3. 代码调整
后端代码需要同步修改：
- Entity 类名：`Order` -> `Orders`
- Mapper 接口：`OrderMapper` -> `OrdersMapper`
- 所有 SQL 语句中的表名
