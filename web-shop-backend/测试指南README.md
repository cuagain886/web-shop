## 🔍 测试重点

### 1. 用户模块
重点关注：
- 注册时用户名、手机号重复校验
- 登录时密码验证
- 用户信息安全性（密码不返回）

### 2. 商品模块
重点关注：
- 分页查询是否正确
- 价格区间筛选是否准确
- 商品状态控制（上下架）

### 3. 购物车模块
重点关注：
- 数量更新是否正确
- 删除操作是否成功
- 购物车统计是否准确

### 4. 订单模块
重点关注：
- 订单创建后库存是否扣减
- 订单状态流转是否正确
- 订单金额计算是否准确

## ❗ 常见问题

### Q: 所有接口都返回404
**A:** 后端服务未启动，执行 `mvn spring-boot:run`

### Q: 接口返回500错误
**A:** 查看后端控制台日志，检查数据库连接和参数

### Q: 新增商品失败
**A:** 确保用户是管理员（role=1），或在数据库手动设置

### Q: 创建订单失败
**A:** 确保购物车有商品，且地址已创建

### Q: Postman无法导入Collection
**A:** 确保文件格式正确（.json），使用Import功能

## 🎓 测试技巧

### 1. 使用环境变量
将常用ID保存为环境变量，方便复用：
```javascript
pm.environment.set("userId", jsonData.data.userId);
pm.environment.set("productId", jsonData.data.id);
```

### 2. 编写测试脚本
在Tests标签添加自动化验证：
```javascript
pm.test("Status code is 200", function () {
    pm.response.to.have.status(200);
});
```

### 3. 使用Collection Runner
批量执行测试，自动生成测试报告。

### 4. 导出测试结果
完成测试后，导出Collection和Environment，分享给团队。

## 🌟 最佳实践

1. **按顺序测试** - 遵循业务流程顺序（注册→登录→浏览→购买）
2. **保存重要数据** - 将ID等关键数据保存到环境变量
3. **验证响应** - 不仅检查状态码，还要验证数据正确性

5. **定期测试** - 代码修改后及时回归测试
