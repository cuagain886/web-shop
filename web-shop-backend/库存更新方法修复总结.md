**修复日期：** 2025-11-04

**状态：** ✅ 已修复

---

## 🎯 问题回顾

### 您的问题

> "为什么 updateStock 的逻辑是将数据库中的 stock 加上 body 中的 stock？"

### 原因分析

**原有设计中的问题：**

1. **Controller 层**：参数名叫 `stock`，看起来是要"设置为某个值"
2. **Service 层**：参数名叫 `quantity`，注释说是"库存变化量"  
3. **SQL 层**：执行的是 `stock = stock + #{quantity}`，实际是"增量更新"

**导致的困惑：**
```
用户输入：{"stock": 120}
用户期望：将库存设置为 120
实际结果：在原库存基础上 +120（如原来50，变成170）❌
```

---

## ✅ 解决方案

### 新增方法：区分两种操作

| 方法 | 用途 | 参数 | SQL |
|------|------|------|-----|
| **setProductStock** | 设置库存（绝对值） | `stock` | `stock = #{stock}` |
| **updateProductStock** | 增减库存（增量） | `quantity` | `stock = stock + #{quantity}` |

## 🎯 两种方法的使用场景

### setProductStock（绝对值）- 管理员使用

**使用场景：**
- ✅ 盘点后修正库存
- ✅ 手动设置初始库存
- ✅ 修正错误的库存数据

**特点：**
- 🔒 **幂等性**：多次执行结果相同
- 📱 **适合界面操作**：输入框输入 120，就设置为 120

**示例：**
```java
productService.setProductStock(1L, 120);  // 设置为 120
```

---

### updateProductStock（增量）- 系统自动使用

**使用场景：**
- ✅ 订单创建时减库存：`updateProductStock(productId, -3)`
- ✅ 订单取消时加库存：`updateProductStock(productId, 3)`
- ✅ 退款时恢复库存：`updateProductStock(productId, 5)`
- ✅ 入库时增加库存：`updateProductStock(productId, 100)`

**特点：**
- ⚠️ **非幂等**：多次执行结果不同
- 🔄 **适合业务逻辑**：基于原有值进行增减

**示例：**
```java
// 订单创建
productService.updateProductStock(productId, -quantity);  // 减库存

// 订单取消
productService.updateProductStock(productId, quantity);   // 加库存
```

### Q1: 还是显示"Content-Type 不支持"？

**A:** 请确保 Postman 中：
1. 选择 **Body** 标签
2. 选择 **raw**

### Q2: 如何增加或减少库存？

**A:** 目前通过 API 只能设置绝对值。如果需要增减，可以：
1. 先查询当前库存
2. 计算新值（如当前 50，想加 10，就设置为 60）
3. 调用设置接口

或者我们可以再添加一个增减库存的 API（如果需要的话）。

### Q3: 系统内部如何增减库存？

**A:** 订单、退款等业务逻辑中，使用 `updateProductStock` 方法：
```java

productService.updateProductStock(productId, -10);  // 减 10
productService.updateProductStock(productId, 20);   // 加 20
```

## 🎓 知识点总结

### 1. 增量更新 vs 绝对值设置

| 操作类型 | SQL | 幂等性 | 适用场景 |
|---------|-----|--------|---------|
| **增量** | `stock = stock + X` | ❌ 非幂等 | 业务流程 |
| **绝对值** | `stock = X` | ✅ 幂等 | 管理操作 |

### 2. API 设计原则

- ✅ **语义清晰**：方法名和参数名要表达清楚意图
- ✅ **最小惊讶**：用户期望的行为就是实际行为
- ✅ **职责分离**：不同的操作用不同的方法

### 3. 命名规范

- `set`：设置为某个值（绝对值）
- `update`：更新（可能是增量）
- `add`：增加
- `reduce`：减少
