# 用户注册字段丢失问题修复

**问题：** 注册时发送了完整的用户信息，但只有部分字段被保存到数据库

**现象：**
- 发送：`nickname: "测试用户2"`，数据库存储：`nickname: "user02"`（用户名）
- 发送：`email`、`avatar`、`gender`、`birthday` 等字段全部丢失
- 发送：`password: "111111"`，数据库存储：`96e79218965eb72c92a549dd5a330112`（MD5加密，正常）

**修复日期：** 2025-11-03  
**修复状态：** ✅ 已完成

---

## 🔍 问题分析

### 原因

**之前的流程：**

1. **前端/Postman发送完整数据：**
```json
{
  "username": "user02",
  "password": "111111",
  "nickname": "测试用户2",
  "phone": "13800138011",
  "email": "user02@example.com",
  "avatar": "/images/avatars/user01.jpg",
  "gender": 1,
  "birthday": "1995-08-15"
}
```

2. **Controller 只提取3个字段：**
```java
userService.register(
    registerDTO.getUsername(),   // ✅
    registerDTO.getPassword(),   // ✅
    registerDTO.getPhone()       // ✅
);
// ❌ nickname、email、avatar、gender、birthday 全部被忽略！
```

3. **Service 强制覆盖 nickname：**
```java
user.setNickname(username);  // ❌ 强制设为用户名，忽略用户输入
```

4. **结果：**
- nickname 被覆盖为 username
- 其他字段全部丢失

### 修复1: 扩展 RegisterDTO

public class RegisterDTO {
    private String username;    // ✅ 必填

### 修复2: 新增 Service 方法

**添加支持完整信息的注册方法：**
```java
// UserService.java
User registerWithFullInfo(User user, String password);

// UserServiceImpl.java
@Override
public User registerWithFullInfo(User user, String password) {
    // 检查用户名和手机号是否已存在
    if (checkUsernameExists(user.getUsername())) {
        throw new IllegalArgumentException("用户名已存在");
    }
    if (checkPhoneExists(user.getPhone())) {
        throw new IllegalArgumentException("手机号已被使用");
    }
    
    // 加密密码
    user.setPassword(encryptPassword(password));
    
    // 设置默认值（仅当没有提供时）
    if (user.getNickname() == null || user.getNickname().isEmpty()) {
        user.setNickname(user.getUsername());  // ✅ 仅在未提供时使用用户名
    }
    if (user.getRole() == null || user.getRole().isEmpty()) {
        user.setRole("user");
    }
    if (user.getStatus() == null) {
        user.setStatus(1);
    }
    
    // 保存用户
    this.save(user);
    
    return user;
}
```

### 修复3: 更新 Controller

**使用完整信息注册：**
```java
@PostMapping("/register")
public Result<Map<String, Object>> register(@Valid @RequestBody RegisterDTO registerDTO) {
    // 将DTO转换为User实体
    User user = new User();
    user.setUsername(registerDTO.getUsername());
    user.setPhone(registerDTO.getPhone());
    user.setNickname(registerDTO.getNickname());      // ✅ 使用用户提供的昵称
    user.setEmail(registerDTO.getEmail());            // ✅ 使用用户提供的邮箱
    user.setAvatar(registerDTO.getAvatar());          // ✅ 使用用户提供的头像
    user.setGender(registerDTO.getGender());          // ✅ 使用用户提供的性别
    user.setBirthday(registerDTO.getBirthday());      // ✅ 使用用户提供的生日
    
    // 调用新的注册方法
    User registeredUser = userService.registerWithFullInfo(user, registerDTO.getPassword());
    
    // 返回完整信息
    Map<String, Object> data = new HashMap<>();
    data.put("userId", registeredUser.getId());
    data.put("username", registeredUser.getUsername());
    data.put("nickname", registeredUser.getNickname());  // ✅ 返回实际的昵称
    data.put("email", registeredUser.getEmail());
    data.put("avatar", registeredUser.getAvatar());
    data.put("role", registeredUser.getRole());
    
    return Result.success("注册成功", data);
}
```

## 📋 密码加密说明

**为什么密码变成了 `96e79218965eb72c92a549dd5a330112`？**

这是**正常的安全措施**！

```java
// UserServiceImpl.java
private String encryptPassword(String password) {
    // MD5加密（生产环境建议使用BCrypt）
    return DigestUtils.md5DigestAsHex(password.getBytes(StandardCharsets.UTF_8));
}
```

**加密对照：**
| 原始密码 | MD5加密后 |
|---------|----------|
| 111111 | 96e79218965eb72c92a549dd5a330112 |
| 123456 | e10adc3949ba59abbe56e057f20f883e |
| 654321 | c33367701511b4f6020ec61ded352059 |

**为什么要加密？**
1. ✅ 安全性 - 即使数据库泄露，密码也不会暴露
2. ✅ 合规性 - 符合数据安全规范
3. ✅ 行业标准 - 所有正规网站都必须加密存储密码

**登录时如何验证？**
```java
// 将用户输入的密码加密后与数据库比对
if (!user.getPassword().equals(encryptPassword(password))) {
    throw new IllegalArgumentException("用户名或密码错误");
}
```

## 🎯 注意事项

### 1. 字段优先级

**现在的逻辑：**
- ✅ 如果用户提供了 nickname → 使用用户提供的值
- ✅ 如果用户没提供 nickname → 使用 username 作为默认值
- ✅ 其他可选字段同理

### 2. 必填字段

**必填（后端强制校验）：**
- username（3-20字符）
- password（6-20字符）
- phone（手机号格式）

**可选（可以不提供）：**
- nickname（不提供则使用username）
- email
- avatar
- gender
- birthday

### 3. 不允许客户端设置的字段

**这些字段由系统控制：**
- ❌ role - 固定为 "user"（安全考虑）
- ❌ status - 固定为 1（启用）
- ❌ deleted - 固定为 0（未删除）
- ❌ createdTime - 自动填充
- ❌ updatedTime - 自动填充

即使客户端发送了这些字段，也会被忽略或覆盖。
