# 库存更新方法设计说明

**问题发现日期：** 2025-11-04  
**修复内容：** 区分库存的"增量更新"和"绝对值设置"

---

## 🔍 问题背景

用户发现："为什么 updateStock 的逻辑是将数据库中的 stock 加上 body 中的 stock？"

### 原始问题

在原有设计中，存在**语义不一致**的问题：

| 层级 | 参数名 | 实际含义 | 期望含义 |
|------|-------|---------|---------|
| **Controller** | `stock` | 库存数量 | 设置为某个值 |
| **Service** | `quantity` | 库存变化量 | 增减某个量 |
| **Mapper SQL** | `quantity` | `stock + #{quantity}` | 增量更新 |

**示例问题：**
- 管理员输入 `{"stock": 120}`
- 期望：将库存设置为 120
- 实际：在原库存基础上 +120（如果原来是 50，变成 170）

---

## 💡 解决方案

### 新设计：区分两种操作

我们现在提供了**两个独立的方法**：

| 方法 | 用途 | 参数含义 | SQL 操作 |
|------|------|---------|---------|
| **updateProductStock** | 增减库存 | `quantity`（变化量） | `stock = stock + #{quantity}` |
| **setProductStock** | 设置库存 | `stock`（绝对值） | `stock = #{stock}` |

## 📁 修改的文件

### 1. ProductMapper.java

**新增方法：**

```java
/**
 * 设置商品库存（绝对值）
 * 
 * @param productId 商品ID
 * @param stock 库存数量
 */
void setStock(@Param("productId") Long productId, @Param("stock") Integer stock);
```

### 2. ProductMapper.xml

```xml
    SET stock = #{stock}
    WHERE id = #{productId}
```

```java
 * 设置商品库存（绝对值）
 * 
 * @param productId 商品ID
 * @param stock 库存数量
 */

```

### 4. ProductServiceImpl.java

**新增实现：**

```java
@Override
@Transactional(rollbackFor = Exception.class)
public void setProductStock(Long productId, Integer stock) {
    log.info("设置商品库存（绝对值）：productId={}, stock={}", productId, stock);
    baseMapper.setStock(productId, stock);
    log.info("库存设置成功");
}
```

### 5. ProductController.java

**修改前：**
```java
@PutMapping("/{productId}/stock")
public Result<Void> updateStock(...) {
    productService.updateProductStock(productId, updateStockDTO.getStock()); // ❌ 增量更新
}
```

**修改后：**
```java
@PutMapping("/{productId}/stock")
@Operation(summary = "设置库存", description = "管理员设置商品库存为指定值")
public Result<Void> updateStock(...) {
    productService.setProductStock(productId, updateStockDTO.getStock()); // ✅ 绝对值设置
}
```

## 📊 两种方法的详细对比

| 对比项 | updateProductStock (增量) | setProductStock (绝对值) |
|-------|--------------------------|------------------------|
| **参数名** | `quantity` | `stock` |
| **参数含义** | 库存变化量 | 目标库存数量 |
| **SQL 操作** | `stock = stock + #{quantity}` | `stock = #{stock}` |
| **正数效果** | 增加库存 | 设置为该值 |
| **负数效果** | 减少库存 | ❌ 不应使用负数 |
| **使用场景** | 订单、退款、入库、出库 | 盘点、修正、初始化 |
| **幂等性** | ❌ 非幂等 | ✅ 幂等 |

## ⚠️ 幂等性说明

### updateProductStock（增量更新）- 非幂等

**同一个请求执行多次，结果不同：**

```java
// 原库存：100
productService.updateProductStock(1L, 10);  // 第1次：100 → 110
productService.updateProductStock(1L, 10);  // 第2次：110 → 120 ❌
productService.updateProductStock(1L, 10);  // 第3次：120 → 130 ❌
```

**风险：** 需要保证业务层面的幂等性（如订单不能重复扣库存）

---

### setProductStock（绝对值）- 幂等

**同一个请求执行多次，结果相同：**

```java
// 原库存：100
productService.setProductStock(1L, 120);  // 第1次：100 → 120
productService.setProductStock(1L, 120);  // 第2次：120 → 120 ✅
productService.setProductStock(1L, 120);  // 第3次：120 → 120 ✅
```

**优点：** 天然幂等，适合管理员操作

## 📝 命名规范

### 动词选择

| 动词 | 含义 | 操作类型 | 示例 |
|------|------|---------|------|
| **update** | 更新（增量） | `stock = stock + X` | `updateProductStock` |
| **set** | 设置（绝对值） | `stock = X` | `setProductStock` |
| **add** | 增加 | `stock = stock + X` | `addStock` |
| **reduce** | 减少 | `stock = stock - X` | `reduceStock` |

### 参数命名

| 参数名 | 含义 | 取值范围 |
|-------|------|---------|
| **quantity** | 数量、变化量 | 正数/负数 |
| **stock** | 库存（绝对值） | 正数/0 |
| **amount** | 金额 | 正数 |
| **delta** | 变化量（增量） | 正数/负数 |

## 🎓 设计原则

### 1. 单一职责原则

- ✅ `updateProductStock`：只负责增减库存
- ✅ `setProductStock`：只负责设置库存

### 2. 语义清晰原则

- ✅ 方法名明确表达操作类型（update vs set）
- ✅ 参数名明确表达参数含义（quantity vs stock）

### 3. 最小惊讶原则

- ✅ 用户期望"设置为 120"，就真的设置为 120
- ✅ 不要让用户猜测"是设置还是增加"
