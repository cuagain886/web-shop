## 功能概述

### 1. 结算页（Checkout）

**功能特点：**
- ✅ 收货地址管理（默认地址展示、地址切换、地址列表）
- ✅ 商品信息展示（从购物车选中商品）
- ✅ 支付方式选择（微信支付、支付宝）
- ✅ 订单金额汇总（商品总价、运费、实付款）
- ✅ 提交订单功能（创建订单、清空购物车、跳转详情）

**访问路径：** `/checkout`

**权限要求：** 需要登录

### 2. 订单列表页（OrderList）

**功能特点：**
- ✅ 订单状态筛选（全部、待支付、已支付、已发货、已完成、已取消）
- ✅ 订单信息展示（订单号、时间、商品、金额、状态）
- ✅ 订单操作按钮（去支付、取消订单、确认收货、查看详情）
- ✅ 状态数量统计（显示各状态订单数量）

**访问路径：** `/orders`

**权限要求：** 需要登录

### 3. 订单详情页（OrderDetail）

**功能特点：**
- ✅ 订单状态跟踪（可视化流程展示）
- ✅ 收货信息展示（收货人、电话、地址）
- ✅ 商品明细展示（图片、名称、规格、价格、数量、小计）
- ✅ 订单信息展示（订单号、时间、支付方式、物流单号）
- ✅ 费用明细展示（商品总价、运费、实付款）
- ✅ 状态操作按钮（立即支付、取消订单、确认收货）

**访问路径：** `/order/:id`

**权限要求：** 需要登录

#### 核心逻辑

1. **数据获取：**
   - 从购物车Store获取选中商品（`cartStore.checkedItems`）
   - 从地址API获取用户地址列表
   - 自动选择默认地址

2. **表单校验：**
   ```javascript
   // 提交订单前校验
   if (!selectedAddress.value) {
     ElMessage.warning('请选择收货地址')
     return
   }
   if (orderItems.value.length === 0) {
     ElMessage.warning('购物车中没有选中的商品')
     return
   }
   ```

3. **订单提交：**
   ```javascript
   // 准备订单数据
   const orderData = {
     items: [...],      // 商品列表
     totalAmount: xxx,  // 总金额
     address: {...},    // 收货地址
     paymentMethod: 'wechat' // 支付方式
   }
   
   // 创建订单
   const order = await orderStore.createOrder(orderData)
   
   // 清空购物车
   await cartStore.batchDelete(itemIds)
   
   // 跳转订单详情
   router.push(`/order/${order.id}`)
   ```

#### 交互流程

```
进入结算页
    ↓
检查登录状态 → 未登录 → 跳转登录页
    ↓ 已登录
检查选中商品 → 无商品 → 返回购物车
    ↓ 有商品
加载地址列表
    ↓
选择收货地址
    ↓
选择支付方式
    ↓
提交订单确认
    ↓
创建订单成功
    ↓
清空购物车已结算商品
    ↓
跳转订单详情页
```

## 核心逻辑

### 订单生命周期

```
1. 创建订单（Create）
   ↓
2. 待支付（Pending）
   ↓
3. 已支付（Paid）
   ↓
4. 已发货（Shipped）
   ↓
5. 已完成（Completed）

   或

   取消订单 → 已取消（Cancelled）
```

### 订单状态流转

```javascript
// 订单状态流转规则
const statusFlow = {
  pending: {
    allowedActions: ['pay', 'cancel'],
    nextStatus: {
      pay: 'paid',
      cancel: 'cancelled'
    }
  },
  paid: {
    allowedActions: [],
    nextStatus: {
      // 由管理员发货后变为shipped
    }
  },
  shipped: {
    allowedActions: ['confirmReceipt'],
    nextStatus: {
      confirmReceipt: 'completed'
    }
  },
  completed: {
    allowedActions: [],
    nextStatus: {}
  },
  cancelled: {
    allowedActions: [],
    nextStatus: {}
  }
}
```

## 使用指南

### 完整购物流程

#### 1. 购物车 → 结算

```javascript
// 在购物车页面
1. 选择要购买的商品（勾选checkbox）
2. 点击"去结算"按钮
3. 系统检查登录状态
   - 未登录 → 跳转登录页（登录后返回结算页）
   - 已登录 → 直接跳转结算页
```

#### 2. 结算 → 创建订单

```javascript
// 在结算页面
1. 确认商品信息
2. 选择收货地址
   - 默认显示默认地址
   - 可点击"选择其他地址"切换
   - 可点击"添加地址"新增
3. 选择支付方式（微信/支付宝）
4. 确认订单总额
5. 点击"提交订单"
6. 系统创建订单并跳转到订单详情页
7. 购物车中已结算商品自动清空
```

#### 3. 订单详情 → 支付

```javascript
// 在订单详情页面
1. 查看订单状态（待支付）
2. 点击"立即支付"按钮
3. 确认支付（模拟支付，实际1秒延迟）
4. 支付成功，订单状态更新为"已支付"
5. 页面自动刷新显示最新状态
```

#### 4. 查看所有订单

```javascript
// 方式1：Header用户菜单
1. 点击用户名下拉菜单
2. 选择"我的订单"
3. 跳转到订单列表页

// 方式2：直接访问
访问 /orders 路径
```

#### 5. 订单操作

```javascript
// 待支付订单
- 去支付：模拟支付，状态变为"已支付"
- 取消订单：状态变为"已取消"
```

```javascript
const itemIds = orderItems.value.map(item => item.id)
```

### 3. 订单状态更新

订单操作（支付、取消、确认收货）后需要更新Store中的数据：

```javascript
// 更新订单列表
const index = orderList.value.findIndex(o => o.id === orderId)
if (index !== -1) {
  orderList.value[index] = updatedOrder
}

// 更新当前订单
if (currentOrder.value?.id === orderId) {
  currentOrder.value = updatedOrder
}
```
