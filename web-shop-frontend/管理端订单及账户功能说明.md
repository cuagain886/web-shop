# 管理端订单管理和商家账户设置功能说明

## 📋 功能概览

本阶段实现了商家后台的订单管理和账户设置功能，包括：

### 1. 订单管理模块
- ✅ 订单列表页（筛选、搜索、分页）
- ✅ 订单详情页（状态展示、操作按钮）
- ✅ 订单状态管理（取消、发货、备注）

### 2. 商家账户设置模块
- ✅ 个人信息管理
- ✅ 安全设置（密码修改）

---

## 🎯 功能详情

### 一、订单管理模块

#### 1.1 订单列表页 (`AdminOrderList.vue`)

**页面路径**：`/admin/orders`

**主要功能**：
- **搜索功能**：支持按订单号搜索
- **状态筛选**：全部/待支付/已支付/已发货/已完成/已取消
- **时间范围筛选**：按下单时间范围筛选
- **快捷筛选按钮**：快速切换订单状态
- **分页功能**：支持自定义每页显示数量

**数据展示**：
| 字段 | 说明 |
|------|------|
| 订单号 | 唯一标识 |
| 商品信息 | 显示前2件商品图片和名称 |
| 收货人 | 姓名和联系电话 |
| 金额 | 订单总金额 |
| 状态 | 订单当前状态（带颜色标签） |
| 下单时间 | 创建时间 |

**操作按钮**：
- **查看详情**：跳转到订单详情页
- **发货**（仅已支付订单）：弹出物流单号输入框
- **取消订单**（仅待支付订单）：取消并更新状态

**技术实现**：
```vue
// 主要方法
fetchOrders()           // 获取订单列表
handleSearch()          // 搜索
handleReset()           // 重置筛选
quickFilter(status)     // 快捷筛选
handleShipOrder(order)  // 发货
handleCancelOrder(id)   // 取消订单
```

---

#### 1.2 订单详情页 (`AdminOrderDetail.vue`)

**页面路径**：`/admin/orders/:id`

**主要功能**：
1. **订单状态展示**
   - 步骤条显示订单流程
   - 状态标签（待支付/已支付/已发货/已完成/已取消）

2. **收货信息**
   - 收货人姓名
   - 联系电话
   - 收货地址（完整）

3. **商品明细**
   - 商品图片、名称、规格
   - 单价、数量、小计
   - 表格形式展示

4. **订单信息**
   - 订单号
   - 下单时间、支付时间、发货时间
   - 支付方式
   - 物流单号（如已发货）

5. **费用明细**
   - 商品总价
   - 运费（固定为0）
   - 实付款（高亮显示）

6. **订单备注**
   - 显示历史备注
   - 添加新备注（仅已完成订单）

**状态操作**：

| 订单状态 | 可用操作 | 说明 |
|---------|---------|------|
| 待支付 | 取消订单 | 商家手动取消 |
| 已支付 | 订单发货 | 填写物流单号 |
| 已完成 | 添加备注 | 记录订单备注 |

**技术实现**：
```vue
// 主要方法
fetchOrderDetail()     // 获取订单详情
confirmShip()          // 确认发货
handleCancel()         // 取消订单
handleAddNote()        // 添加备注
```

**步骤条逻辑**：
```javascript
const currentStep = computed(() => {
  const statusSteps = {
    'pending': 1,    // 提交订单
    'paid': 2,       // 支付成功
    'shipped': 3,    // 商品发货
    'completed': 4,  // 确认收货
    'cancelled': 0   // 已取消
  }
  return statusSteps[order.value.status] || 0
})
```

#### 2.2 安全设置页 (`AdminSecurity.vue`)

**页面路径**：`/admin/security`

**主要功能**：

**密码修改表单**：
- 当前密码输入
- 新密码输入（6-20位）
- 确认新密码输入
- 修改密码按钮

**密码规则**：
1. 密码长度为6-20位
2. 新密码不能与旧密码相同
3. 两次输入的新密码必须一致
4. 建议使用字母、数字和符号的组合

**表单验证**：
```javascript
// 自定义验证器
const validateNewPassword = (rule, value, callback) => {
  if (value === '') {
    callback(new Error('请输入新密码'))
  } else if (value.length < 6 || value.length > 20) {
    callback(new Error('密码长度为6-20位'))
  } else if (value === passwordForm.oldPassword) {
    callback(new Error('新密码不能与旧密码相同'))
  } else {
    callback()
  }
}

const validateConfirmPassword = (rule, value, callback) => {
  if (value === '') {
    callback(new Error('请再次输入新密码'))
  } else if (value !== passwordForm.newPassword) {
    callback(new Error('两次输入的密码不一致'))
  } else {
    callback()
  }
}
```

**修改成功后的操作**：
1. 显示成功提示
2. 清空表单
3. 1.5秒后自动退出登录
4. 跳转到商家登录页

**Mock实现**：
```javascript
// 验证旧密码（从localStorage获取）
const savedPassword = localStorage.getItem('merchant_password')
if (savedPassword && savedPassword !== passwordForm.oldPassword) {
  ElMessage.error('当前密码不正确')
  return
}

// 保存新密码
localStorage.setItem('merchant_password', passwordForm.newPassword)
```

## 🔒 权限控制

### 路由守卫

所有管理端路由都包含以下权限验证：

```javascript
meta: {
  requiresAuth: true,       // 需要登录
  requiresMerchant: true    // 需要商家角色
}
```

**验证逻辑**（`router/index.js`）：
```javascript
// 检查是否登录
if (to.meta.requiresAuth && !userStore.isLoggedIn) {
  // 跳转到商家登录页
  next('/admin/login')
  return
}

// 检查是否是商家账号
if (to.meta.requiresMerchant && userStore.userInfo?.role !== 'merchant') {
  ElMessage.error('无权访问，仅限商家账号')
  next('/admin/login')
  return
}
```
