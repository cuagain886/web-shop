# 路由配置说明

## 📋 概述

本项目采用 Vue Router 4，实现了用户端和管理端的分离式路由设计。

---

## 🚀 路由结构

### 用户端路由（前台）
- **首页**: `/` - 商品展示首页
- **商品详情**: `/product/:id` - 商品详情页
- **购物车**: `/cart` - 购物车页面
- **订单确认**: `/checkout` - 订单确认页（需登录）
- **我的订单**: `/orders` - 订单列表（需登录）
- **订单详情**: `/order/:id` - 订单详情（需登录）
- **用户登录**: `/login` - 用户登录页
- **用户注册**: `/register` - 用户注册页

### 管理端路由（后台）
- **商家登录**: `/admin/login` - 商家登录页
- **管理首页**: `/admin` 或 `/admin/dashboard` - 数据概览（需商家权限）
- **商品管理**:
  - `/admin/products` - 商品列表
  - `/admin/products/add` - 新增商品
  - `/admin/products/edit/:id` - 编辑商品
  - `/admin/categories` - 分类管理
- **订单管理**:
  - `/admin/orders` - 订单列表
  - `/admin/orders/:id` - 订单详情
- **系统管理**:
  - `/admin/operation-log` - 操作日志
  - `/admin/profile` - 个人信息
  - `/admin/security` - 安全设置

## 🔐 权限控制

### 路由元信息（Meta）

```javascript
{
  requiresAuth: true,      // 需要登录
  requiresMerchant: true,  // 需要商家权限
  title: '页面标题'        // 页面标题
}
```

### 权限验证流程

#### 1. 用户端权限
```
访问需要登录的页面
  ↓
未登录？→ 跳转到 /login?redirect=目标页面
  ↓
已登录 → 允许访问
```

#### 2. 管理端权限
```
访问管理端页面
  ↓
未登录？→ 跳转到 /admin/login?redirect=目标页面
  ↓
已登录但不是商家？→ 提示权限不足，跳转到 /admin/login
  ↓
商家登录 → 允许访问
```

## 🎯 登录后跳转规则

### 商家登录（AdminLogin）

```javascript
// 登录成功后
if (redirect参数存在 && redirect以/admin开头 && redirect不是登录页) {
  跳转到 redirect 指定的页面
} else {
  跳转到 /admin/dashboard（管理首页）
}
```

**示例**：
- 直接访问 `/admin/login` → 登录后跳转到 `/admin/dashboard`
- 访问 `/admin/products` 被拦截 → 登录后回到 `/admin/products`

### 用户登录（Login）

```javascript
// 登录成功后
const redirect = route.query.redirect || '/'
跳转到 redirect 指定的页面
```

**示例**：
- 直接访问 `/login` → 登录后跳转到 `/`（首页）
- 访问 `/checkout` 被拦截 → 登录后回到 `/checkout`

## 🛡️ 路由守卫逻辑

### 全局前置守卫（beforeEach）

```javascript
router.beforeEach((to, from, next) => {
  // 1. 设置页面标题
  document.title = to.meta.title + (isAdmin ? ' - 商家后台' : ' - 购物平台')
  
  // 2. 检查是否需要登录
  if (to.meta.requiresAuth && !isLoggedIn) {
    // 跳转到对应的登录页
  }
  
  // 3. 检查是否需要商家权限
  if (to.meta.requiresMerchant && role !== 'merchant') {
    // 拒绝访问
  }
  
  // 4. 已登录用户访问登录页 → 重定向
  if (已登录 && 访问登录页) {
    // 商家 → /admin/dashboard
    // 用户 → /
  }
  
  // 5. 允许通过
  next()
})
```

## 📍 重定向规则

### 1. 访问 `/admin` 根路径
```javascript
{
  path: '/admin',
  redirect: '/admin/dashboard'  // 自动重定向到管理首页
}
```

### 2. 已登录用户访问登录页
```javascript
// 商家已登录，访问 /admin/login
→ 重定向到 /admin/dashboard

// 用户已登录，访问 /login
→ 重定向到 /
```

### 3. 404 页面
```javascript
{
  path: '/:pathMatch(.*)*',
  redirect: '/'  // 未匹配的路径重定向到首页
}
```

## 🔄 路由跳转方式

### 1. 声明式导航

```vue
<!-- router-link -->
<router-link to="/admin/dashboard">管理首页</router-link>
<router-link :to="{ name: 'AdminDashboard' }">管理首页</router-link>
<router-link :to="{ path: '/product/' + id }">商品详情</router-link>
```

### 2. 编程式导航

```javascript
// 跳转到指定路径
router.push('/admin/dashboard')

// 命名路由跳转
router.push({ name: 'AdminDashboard' })

// 带参数跳转
router.push({ 
  path: '/product/123',
  query: { from: 'search' }
})

// 后退
router.back()

// 替换当前历史记录
router.replace('/admin/login')
```

## 📝 使用示例

### 示例 1：商家登录流程

```
1. 访问 http://localhost:5173/admin/products
   ↓ 未登录，被路由守卫拦截
2. 重定向到 /admin/login?redirect=/admin/products
   ↓ 输入商家账号密码
3. 登录成功，解析 redirect 参数
   ↓ 跳转到 /admin/products
4. 成功访问商品列表页
```

### 示例 2：用户下单流程

```
1. 浏览首页 / → 点击商品 → /product/1
   ↓ 加入购物车
2. 访问购物车 /cart → 点击结算
   ↓ 跳转到 /checkout
3. 未登录，被路由守卫拦截
   ↓ 重定向到 /login?redirect=/checkout
4. 登录成功 → 返回 /checkout
   ↓ 提交订单
5. 跳转到订单详情 /order/:id
```

### 示例 3：直接访问 /admin

```
1. 访问 http://localhost:5173/admin
   ↓ adminRoutes 配置了 redirect
2. 自动重定向到 /admin/dashboard
   ↓ 检查权限
3. 未登录 → 跳转 /admin/login
   或
   已登录商家 → 显示数据概览页
```

## 🚨 常见问题

### Q1: 登录后没有跳转到管理页面？
**A**: 检查以下几点：
1. 确认使用的是商家账号（role: 'merchant'）
2. 检查 localStorage 中是否保存了正确的用户信息
3. 查看浏览器控制台是否有路由错误

### Q2: 访问管理页面一直跳转到登录页？
**A**: 
1. 确认已成功登录（userStore.isLoggedIn 为 true）
2. 确认账号角色为 'merchant'
3. 清除浏览器缓存和 localStorage 后重新登录

### Q3: 如何添加不需要权限的管理端页面？
**A**: 在路由配置中不添加 requiresAuth 和 requiresMerchant：
```javascript
{
  path: '/admin/public-page',
  component: () => import('@/pages/admin/PublicPage.vue'),
  meta: { title: '公开页面' }  // 不添加权限要求
}
```

### Q4: redirect 参数没有生效？
**A**: 检查以下几点：
1. AdminLogin.vue 中是否正确处理了 redirect 参数
2. redirect 参数是否以 /admin 开头
3. redirect 参数不能是 /admin/login 本身

## 📌 注意事项

1. **路由懒加载**: 所有页面组件都使用 `() => import()` 动态导入，提高首屏加载速度

2. **权限隔离**: 商家和用户使用不同的登录页和权限系统，互不干扰

3. **URL 参数**: 使用 `route.query` 获取 URL 参数，使用 `route.params` 获取路径参数

4. **路由模式**: 使用 History 模式（createWebHistory），URL 更美观，但需要服务器配置

5. **滚动行为**: 每次路由跳转后自动滚动到页面顶部
