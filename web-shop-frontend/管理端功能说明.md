## 核心逻辑

### 权限控制流程

```
用户访问管理端页面
    ↓
检查是否登录 → 未登录 → 跳转 /admin/login
    ↓ 已登录
检查用户角色
    ↓
role === 'merchant' → 允许访问
    ↓
role === 'user' → 拒绝访问 → 提示"无权限"
```

### 路由守卫实现

```javascript
router.beforeEach((to, from, next) => {
  const userStore = useUserStore()
  
  // 检查是否需要登录
  if (to.meta.requiresAuth) {
    if (!userStore.isLoggedIn) {
      if (to.meta.requiresMerchant) {
        next({ path: '/admin/login', query: { redirect: to.fullPath } })
      } else {
        next({ path: '/login', query: { redirect: to.fullPath } })
      }
      return
    }
    
    // 检查商家权限
    if (to.meta.requiresMerchant) {
      if (userStore.userInfo?.role !== 'merchant') {
        ElMessage.error('无权访问，仅限商家账号')
        next('/admin/login')
        return
      }
    }
  }
  
  next()
})
```

### 商品状态管理

```javascript
// 商品状态枚举
const ProductStatus = {
  PENDING: 'pending',   // 待上架
  ACTIVE: 'active',     // 上架中
  INACTIVE: 'inactive'  // 已下架
}

// 状态操作逻辑
const updateProductStatus = (id, newStatus) => {
  const product = products.find(p => p.id === id)
  
  // 状态流转规则
  // pending → active (上架)
  // active → inactive (下架)
  // inactive → active (重新上架)
  
  product.status = newStatus
  product.updatedAt = new Date().toISOString()
}
```

## 注意事项

### 1. 权限验证

- ✅ 所有管理端页面都需要登录
- ✅ 所有管理端页面都需要商家角色
- ✅ 普通用户账号无法访问管理端
- ✅ 未登录自动跳转到商家登录页

### 2. 数据校验

**商品表单：**
- 商品名称：必填，最大100字符
- 分类：必填
- 价格：必填，大于等于0
- 库存：必填，大于等于0
- 图片：至少1张，最多5张

**分类表单：**
- 分类名称：必填，最大50字符
- 层级：最多二级
- 删除限制：不能有子分类或商品

### 3. Mock数据限制

当前使用Mock数据，以下功能仅为模拟：
- ❌ 真实的图片上传（返回占位图URL）
- ❌ 真实的数据统计（使用固定值）
- ❌ 订单管理功能（待开发）
- ❌ 用户管理功能（待开发）

### 4. 后端接口预留

所有API函数都预留了后端接口调用：

```javascript
export function addProduct(data) {
  if (MOCK_ENABLED) {
    // Mock逻辑
  }
  
  // 后端接口（将来启用）
  return request({
    url: '/admin/products',
    method: 'post',
    data
  })
}
```

要切换到真实后端：
1. 将各API文件中的`MOCK_ENABLED`设置为`false`
2. 确保后端接口已部署
3. 调整`vite.config.js`中的代理配置
