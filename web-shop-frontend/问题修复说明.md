# 商家后台空白页面问题修复说明

## 🐛 问题描述

商家登录后台后，页面显示空白，无法正常显示数据概览页面。

## 🔍 问题原因

在 `src/api/statistics.js` 文件中，需要导入 `ORDER_STORAGE_KEY` 常量：

```javascript
import { ORDER_STORAGE_KEY } from './order'
```

但是在 `src/api/order.js` 文件中，这个常量没有被导出（export），导致导入失败，进而导致整个统计模块加载失败，最终导致管理后台数据概览页面无法渲染。

## ✅ 修复方案

在 `src/api/order.js` 文件中，将 `ORDER_STORAGE_KEY` 常量导出：

**修改前：**
```javascript
const ORDER_STORAGE_KEY = 'mock_order_data' // localStorage存储key
```

**修改后：**
```javascript
export const ORDER_STORAGE_KEY = 'mock_order_data' // localStorage存储key（导出供其他模块使用）
```

## 📝 相关文件

- `src/api/order.js` - 订单API（已修复）
- `src/api/statistics.js` - 统计API（依赖order.js）
- `src/pages/admin/AdminDashboard.vue` - 数据概览页面

## 💡 预防措施

为了避免类似问题，建议：

1. **使用 ESLint**：配置 ESLint 检查未导出的导入
2. **TypeScript**：使用 TypeScript 可以在编译时发现此类错误
3. **代码审查**：在添加新的导入时，确保相应的导出存在
