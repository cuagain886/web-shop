# 快速开始指南

## 环境要求

- Node.js >= 16.0.0
- npm >= 8.0.0

## 安装步骤

### 1. 克隆项目（如果是从仓库获取）

```bash
git clone <repository-url>
cd WebShop/FrontEnd/web-shop-frontend
```

### 2. 安装依赖

```bash
npm install
```

安装过程可能需要几分钟，请耐心等待。

### 3. 启动开发服务器

```bash
npm run dev
```

启动成功后，控制台会显示：

```
  VITE v7.x.x  ready in xxx ms

  ➜  Local:   http://localhost:5173/
  ➜  Network: http://xxx.xxx.xxx.xxx:5173/
```

### 4. 在浏览器中打开

访问 `http://localhost:5173/` 即可看到项目。

## 项目命令

```bash
# 启动开发服务器（热重载）
npm run dev

# 构建生产版本
npm run build

# 预览生产版本
npm run preview

# 代码格式化
npm run format

# 代码检查
npm run lint
```

## 开发提示

### 热重载

开发服务器启动后，修改代码会自动重新加载，无需手动刷新浏览器。

### 控制台日志

打开浏览器开发者工具（F12），可以在控制台看到详细的日志信息：

- 应用初始化过程
- 组件加载状态
- 数据加载情况
- 路由跳转信息

### 调试建议

1. **使用 Vue DevTools**
   - 安装 Vue DevTools 浏览器扩展
   - 可以查看组件树、状态、路由等

2. **查看网络请求**
   - 打开浏览器 Network 面板
   - 查看 API 请求（当前是模拟数据）

3. **性能分析**
   - 使用 Performance 面板分析渲染性能
   - 使用 Lighthouse 进行整体评估

## 常见问题

### 1. 端口被占用

如果 5173 端口被占用，Vite 会自动尝试下一个端口（5174、5175...）

### 2. 依赖安装失败

```bash
# 清除缓存
npm cache clean --force

# 删除 node_modules 和 package-lock.json
rm -rf node_modules package-lock.json

# 重新安装
npm install
```

### 3. 页面样式异常

```bash
# 清除浏览器缓存
# 或使用 Ctrl+Shift+R 强制刷新
```

### 4. 热重载不工作

```bash
# 重启开发服务器
# 按 Ctrl+C 停止，然后重新运行
npm run dev
```
